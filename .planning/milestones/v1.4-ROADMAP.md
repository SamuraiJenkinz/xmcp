# Milestone v1.4: Message Trace & Feedback Analytics

**Status:** SHIPPED 2026-04-06
**Phases:** 26-28
**Total Plans:** 6

## Overview

Add a message trace MCP tool for email delivery tracking (Get-MessageTraceV2) and three feedback analytics MCP tools that query the existing SQLite feedback table directly from the MCP server — bringing the total tool count from 17 to 21. Backend-only milestone with zero frontend changes.

## Phases

### Phase 26: Message Trace Tool

**Goal**: Users can track email delivery status through conversational queries — answering "did my email arrive?" without PowerShell access
**Depends on**: Nothing (first phase of v1.4; uses proven Exchange tool pattern)
**Plans**: 2 plans

Plans:
- [x] 26-01: RBAC verification and Get-MessageTraceV2 tool handler implementation
- [x] 26-02: System prompt disambiguation and tool registration finalization

**Details:**
- get_message_trace MCP tool backed by Get-MessageTraceV2
- Sender/recipient validation, UPN format enforcement
- 10-day max date range, default last 24 hours
- PII-safe subject truncation (30 char limit)
- Result count capping (default 100, max 1000)
- Routing detail (FromIP, ToIP, connector name)
- RBAC verified: Atlas in Organization Management (includes Message Tracking)
- System prompt rules 15-18 for intent-based disambiguation vs check_mail_flow

### Phase 27: Feedback Analytics Foundation

**Goal**: Users can query aggregate feedback data through conversation — vote counts, satisfaction trends, and detailed negative feedback review
**Depends on**: Phase 26 (sequential ordering; no code dependency)
**Plans**: 2 plans

Plans:
- [x] 27-01: ATLAS_DB_PATH plumbing and read-only SQLite connection pattern
- [x] 27-02: get_feedback_summary and get_low_rated_responses tool handlers

**Details:**
- New module: exchange_mcp/feedback_analytics.py (isolated from Exchange handlers)
- _open_ro: read-only sqlite3 URI connection with Windows path encoding
- get_feedback_summary: total/up/down votes, satisfaction percentage, daily trend with zero-fill
- get_low_rated_responses: thumbs-down entries with thread names and optional comments
- asyncio.to_thread wraps all sqlite3 blocking I/O
- ATLAS_DB_PATH environment variable for independent database configuration
- System prompt rules 19-22 for routing and privacy boundary

### Phase 28: Tool Correlation & Analytics Completion

**Goal**: Users can identify which Exchange tools produce the worst user experience and the AI presents all analytics results conversationally
**Depends on**: Phase 27 (builds on SQLite access pattern and feedback_analytics.py module)
**Plans**: 2 plans

Plans:
- [x] 28-01: get_feedback_by_tool handler with message-to-tool correlation logic
- [x] 28-02: System prompt analytics guidance and presentation rules

**Details:**
- _find_tool_names: backward-walk correlation through message history (tool_calls + legacy function_call)
- Fan-out vote attribution: multi-tool messages assign vote to each tool independently
- Two-mode handler: breakdown (per-tool satisfaction, worst-first, low-confidence flagged) and drill-down (thumbs-down examples for named tool)
- Low-confidence threshold at < 5 votes (flagged and sorted last)
- System prompt rules 23-26: routing, presentation (executive summary tone), empty-state fallback

## Milestone Summary

**Key Decisions:**

- Use Get-MessageTraceV2 (not deprecated V1) for Exchange Online message tracking
- Explicit negative rule for check_mail_flow misrouting (positive rules alone insufficient)
- No PRAGMAs in _open_ro (database already has WAL; read-only connections cannot write)
- asyncio.to_thread for all sqlite3 I/O (prevents blocking MCP event loop)
- ATLAS_DB_PATH separate from CHAT_DB_PATH (allows independent configuration)
- Fan-out vote attribution for multi-tool messages (preserves per-tool accuracy)
- Low-confidence flag at < 5 votes, sorted last not excluded
- Two-mode handler via tool_name param (reduces tool count)
- Rules 25 lettered sub-points (a-d) for presentation concerns

**Issues Resolved:**

- RBAC verification: Atlas already in Organization Management (includes Message Tracking) — no changes needed
- Tool disambiguation: intent-based routing with negative guidance prevents get_message_trace / check_mail_flow confusion

**Issues Deferred:**

- Historical message trace beyond 10 days via Start-HistoricalSearch (async, up to 90 days) — TRACE-11
- Entra ID registered device lookup alongside Exchange ActiveSync — TRACE-12
- Admin-only web dashboard for feedback visualization — FBAN-12
- Feedback export to CSV/JSON — FBAN-13
- Feedback correlation with conversation length/complexity — FBAN-14

**Technical Debt Incurred:**

- server.py:155 handle_list_tools docstring says "18 tools" (should be 21) — cosmetic
- tests/test_server.py:5,12 module docstring lists "17 tools" (should be 21) — cosmetic
