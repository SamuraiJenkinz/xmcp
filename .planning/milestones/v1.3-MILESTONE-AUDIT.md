## Tech Debt

### Phase 25: Motion Animations

- **Sidebar CSS transition lacks prefers-reduced-motion override**
  - File: `frontend/src/index.css`, line 181
  - Issue: `.sidebar { transition: width 225ms ease-in-out }` has no `@media (prefers-reduced-motion: reduce)` rule
  - Impact: Users with OS reduce-motion still see 225ms sidebar collapse/expand animation
  - Severity: Low — CSS polish, not functional
  - Fix: Add `@media (prefers-reduced-motion: reduce) { .sidebar { transition: none; } }` consistent with `.feedback-scale-btn` pattern at lines 1044-1047
