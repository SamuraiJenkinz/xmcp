# Requirements: Exchange Infrastructure MCP Server

**Defined:** 2026-03-19
**Core Value:** Any colleague with appropriate access can interrogate Exchange infrastructure through conversational queries against live environment data

## v1 Requirements

Requirements for initial working demo. Each maps to roadmap phases.

### Exchange Client

- [x] **EXCL-01**: Async PowerShell subprocess runner using asyncio.create_subprocess_exec with per-call PSSession lifecycle (create + destroy each invocation) — *Validated v1.0*
- [x] **EXCL-02**: Interactive auth (browser popup) as default, certificate-based auth as optional fallback (Kerberos deferred to v2) — *Validated v1.0*
- [x] **EXCL-03**: DNS resolver for DMARC/SPF lookups via dnspython (no PowerShell dependency) — *Validated v1.0*
- [x] **EXCL-04**: Structured JSON output parsing with ConvertTo-Json -Depth 10 and explicit Select-Object field selection per tool — *Validated v1.0*

### MCP Server Infrastructure

- [x] **MCPS-01**: Tool registration via official mcp SDK with stdio transport — *Validated v1.0*
- [x] **MCPS-02**: Structured error handling — isError: true on all failure paths with sanitized error messages — *Validated v1.0*
- [x] **MCPS-03**: Tool descriptions optimized for LLM tool selection (<800 characters each) — *Validated v1.0*
- [x] **MCPS-04**: All logging to stderr only — zero stdout pollution in server.py — *Validated v1.0*

### Mailbox Tools

- [x] **MBOX-01**: get_mailbox_stats — returns size, quota, last logon, database placement for a single mailbox — *Validated v1.0*
- [x] **MBOX-02**: search_mailboxes — filters mailboxes by database, type, or display name — *Validated v1.0*
- [x] **MBOX-03**: get_shared_mailbox_owners — returns full access, send-as, and send-on-behalf delegates — *Validated v1.0*

### DAG / Database Tools

- [x] **DAGD-01**: list_dag_members — lists DAG member servers with operational status and active database count — *Validated v1.0*
- [x] **DAGD-02**: get_dag_health — full replication health report including copy/replay queue lengths and content index — *Validated v1.0*
- [x] **DAGD-03**: get_database_copies — returns all copies of a database across DAG members with activation preferences — *Validated v1.0*

### Mail Flow Tools

- [x] **FLOW-01**: check_mail_flow — traces routing path between sender and recipient, connector resolution, TLS requirements — *Validated v1.0*
- [x] **FLOW-02**: get_transport_queues — returns queue depths across all transport servers, flags backlogs over threshold — *Validated v1.0*
- [x] **FLOW-03**: get_smtp_connectors — full inventory of send and receive connectors with auth and TLS configuration — *Validated v1.0*

### Security Tools

- [x] **SECU-01**: get_dkim_config — DKIM signing configuration per domain, selector names, CNAME records — *Validated v1.0*
- [x] **SECU-02**: get_dmarc_status — live DNS lookup of DMARC, SPF records with parsed policy values — *Validated v1.0*
- [x] **SECU-03**: check_mobile_devices — ActiveSync device partnerships, access state, last sync, wipe history — *Validated v1.0*

### Hybrid Tools

- [x] **HYBR-01**: get_hybrid_config — full hybrid topology: org relationships, federation trust, connector mapping — *Validated v1.0*
- [~] **HYBR-02**: ~~get_migration_batches~~ — *Removed: MMC does not use migration batches; tool removed in Phase 6 plan 06-01*
- [x] **HYBR-03**: get_connector_status — validates hybrid connector health with TLS certificate validation — *Validated v1.0*

### Chat Application

- [x] **CHAT-01**: Azure AD / Entra ID SSO via MSAL auth code flow (corporate identity login) — *Validated v1.0*
- [x] **CHAT-02**: Azure OpenAI tool-calling loop — detect tool_calls, route to MCP server, append results, second completion call — *Validated v1.0*
- [x] **CHAT-03**: SSE streaming of final AI response to browser in real-time — *Validated v1.0*
- [x] **CHAT-04**: Context window management — tiktoken token counting, conversation pruning within 128K token limit — *Validated v1.0*

### User Interface

- [x] **UIUX-01**: Conversation history persisting across sessions (SQLite-backed, scoped to authenticated user) — *Validated v1.0*
- [x] **UIUX-02**: Multiple conversation threads with sidebar navigation (create, switch, delete) — *Validated v1.0*
- [x] **UIUX-03**: Tool visibility panel — collapsible panel showing tool name, parameters, and raw Exchange result — *Validated v1.0*
- [x] **UIUX-04**: Copy/export responses for reports or tickets — *Validated v1.0*
- [x] **UIUX-05**: Conversation auto-naming from first query text — *Validated v1.0*
- [x] **UIUX-06**: Loading indicators — "Querying Exchange..." status during tool execution (2-4s expected) — *Validated v1.0*
- [x] **UIUX-07**: Keyboard shortcuts (Ctrl+Enter to send, Esc to cancel) — *Validated v1.0*
- [x] **UIUX-08**: Dark mode toggle — *Validated v1.0*
