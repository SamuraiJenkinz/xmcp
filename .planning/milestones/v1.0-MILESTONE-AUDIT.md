## E2E Flow Results

### Flow 1: First-time login → Question → Answer with tool panel
**Status: COMPLETE**

Login → Azure AD → session → sidebar (empty) → send message → bouncing dots → tool SSE events → panels rendered → text streaming → finalize → copy button → auto-naming → sidebar update. All steps verified against code.

### Flow 2: Return visit → Switch threads → Copy response
**Status: PARTIAL**

Login via session → sidebar loads → thread switch → messages load → **but**: historical messages render as plain text without copy buttons or tool panels. Only messages received during the current session have the full UX.

**Root cause:** `switchThread()` renders `<p>` elements directly, not via `createAssistantMessage()`. Tool events are not persisted to SQLite (only assistant text content stored).

### Flow 3: Cancel mid-stream → Dark mode → New thread
**Status: COMPLETE**

Esc → AbortController.abort() → partial text preserved → "[response cancelled]" marker. Dark mode toggle → localStorage persistence → flash prevention on reload. New thread → POST /api/threads → welcome message.

## Tech Debt

### Phase 1: Exchange Client Foundation
- **2 human-needed verifications**: Run `scripts/verify_exchange.py` against live Exchange Online and `uv run pytest tests/test_ps_runner.py` on Windows with PowerShell. Cannot be verified without live environment.

### Phase 7: Chat App Core
- **CHATGPT_ENDPOINT operational gap**: Not included in `secrets.py` fallback dict. Must be set as a separate environment variable. Works correctly if env var is present — not a code bug, but an operational gotcha during deployment.

### Phase 8/9: Historical Message UX
- **Copy button missing on historical messages**: `switchThread()` renders plain `<p>` elements, not `createAssistantMessage()` with copy infrastructure. Only live-streamed messages get copy buttons.
- **Tool panels not persisted**: `tool_events` are ephemeral SSE state, not stored in SQLite. When switching to a historical thread, tool panels are permanently lost. Only the final assistant text response is persisted.

**Impact:** Medium. Core value (ask Exchange questions, get answers) works fully. The polish features (copy, tool panels) only apply to the current session's messages. This is a known architectural limitation, not a regression.

## Conclusion

All 34 active v1 requirements are satisfied. All 9 phases verified against codebase. Cross-phase integration is solid with 18/18 connections wired correctly. The tech debt items are non-blocking — they represent UX polish gaps on historical message display and operational deployment notes, not functional breaks.
