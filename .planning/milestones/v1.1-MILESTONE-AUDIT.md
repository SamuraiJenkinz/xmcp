## Tech Debt

### Phase 11: MCP Tools + Photo Proxy (4 items)

1. **Test regression: description phrasing** — `search_colleagues` description uses "Use for" not "Use when", failing `test_all_descriptions_contain_use_when`. Fix: 1 line change in tools.py:374.

2. **Test regression: missing example query** — `get_colleague_profile` description has no single-quoted example query, failing `test_all_descriptions_contain_example_query`. Fix: 1 line change in tools.py:391-394.

3. **Test regression: stale tool count** — `test_list_tools_returns_all_15` asserts `len == 15` but there are now 17 tools. Fix: 1 line change in tests/test_server.py.

4. **Misleading schema description** — `get_colleague_profile` parameter `user_id` is described as "The colleague's user ID from search results" but `search_colleagues` only returns email. Graph accepts email as a path segment, so it works at runtime, but the schema description is misleading for LLM guidance. Fix: update description to say "email address or Graph object ID".

### Phase 10: Graph Client Foundation (2 items)

5. **Dead code** — `get_user_photo_bytes()` in graph_client.py is defined but never called. Only `get_user_photo_96()` is used by the photo proxy. Low priority: may be useful for future non-96px sizes.

6. **Human verification pending** — Admin consent (GRAPH-02) and role claims in live JWT require manual verification on the deployed server. Not a code gap; inherent constraint of testing against Azure AD.

### Total: 6 items across 2 phases
