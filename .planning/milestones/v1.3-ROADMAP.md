# Milestone v1.3: Access Control, Feedback, Search, Export, Animations

**Status:** SHIPPED 2026-04-02
**Phases:** 21-25
**Total Plans:** 9

## Overview

Gate access to authorized IT engineers via Azure AD App Roles, add per-message thumbs up/down feedback, thread search in the sidebar, Markdown export for tickets, and motion entrance animations consistent with the Copilot aesthetic established in v1.2.

## Phases

### Phase 21: App Role Access Control

**Goal**: Only authorized IT engineers (holding the Atlas.User App Role) can access the application — all other authenticated users see a Fluent 2 access denied experience instead of the chat interface.
**Depends on**: Phase 20 (v1.2 shipped React app and AuthContext)
**Plans**: 2 plans

Plans:
- [x] 21-01: role_required decorator, session roles extraction, 403/401 response paths (backend)
- [x] 21-02: AuthContext 403 state, AccessDenied component, AuthGuard routing (frontend)

**Details:**
- Created `role_required` decorator with REQUIRED_ROLE = "Atlas.User" constant
- Structured 403 JSON with error, message, required_role, upn fields
- All 9 protected routes migrated from login_required to role_required
- /api/me extended to return roles list for authorized users
- AuthStatus discriminated union (loading/authenticated/unauthenticated/forbidden/error) replaces boolean
- AccessDenied Fluent 2 card with ShieldKeyhole icon, UPN copy button, mailto admin link
- SSE 401/403 triggers page reload for AuthGuard re-evaluation

---

### Phase 22: Per-Message Feedback

**Goal**: IT engineers can vote thumbs up or down on any assistant message after it finishes streaming, and votes persist to SQLite against the user's identity for future analytics.
**Depends on**: Phase 21 (role_required must be applied to feedback endpoints)
**Plans**: 2 plans

Plans:
- [x] 22-01: feedback SQLite table schema, feedback Blueprint (POST/DELETE endpoints)
- [x] 22-02: AssistantMessage feedback buttons, optimistic state, ARIA live region, Popover for comment

**Details:**
- feedback table with UNIQUE(thread_id, assistant_message_idx, user_id), CHECK(vote IN ('up','down')), ON DELETE CASCADE
- migrate_db() idempotent startup migration — existing databases gain feedback table automatically
- feedback_bp Blueprint with GET/POST/DELETE endpoints, all protected by role_required
- FeedbackButtons component with ThumbLike/ThumbDislike icons via bundleIcon
- Optimistic dispatch + async backend pattern for instant UI response
- Popover with optional freetext comment (500 char limit) on thumbs-down
- ARIA live region announces "Feedback submitted" to screen readers

---

### Phase 23: Thread Search

**Goal**: Users can find threads instantly by typing in the sidebar search box, and can perform full-text search across message content when a thread name alone is not enough.
**Depends on**: Phase 21 (search endpoint must be protected by role_required)
**Plans**: 2 plans

Plans:
- [x] 23-01: FTS5 virtual table, sync triggers, backfill, search endpoint with user scoping
- [x] 23-02: SearchInput component, client-side title filter, FTS5 results display, Ctrl+K shortcut

**Details:**
- FTS5 threads_fts virtual table with unicode61 tokenizer, 3 sync triggers, idempotent backfill
- GET /api/threads/search with role_required, user_id JOIN scoping, graceful malformed query handling
- _build_fts5_query quotes tokens to neutralise FTS5 operators
- SearchInput with Fluent SearchBox, CounterBadge, snippet display
- Two-tier search: instant client-side title filter + 300ms debounced backend FTS5
- Ctrl+K shortcut focuses search from anywhere, expanding collapsed sidebar first

---

### Phase 24: Conversation Export

**Goal**: IT engineers can download the active thread as a Markdown file for pasting into Jira/incident reports, with all tool call data included.
**Depends on**: Phase 21 (export endpoint must verify thread ownership and require role)
**Plans**: 1 plan

Plans:
- [x] 24-01: Export utilities (Markdown renderer, slugify, Blob download), ExportMenu component, Header integration

**Details:**
- Pure utility modules: exportMarkdown, slugify, downloadBlob (no React imports)
- ExportMenu Fluent component with ArrowDownloadRegular icon and Markdown format option
- Tool panels appear before assistant content in Markdown output (mirrors UI order)
- Filename: slugified-thread-name-YYYY-MM-DD.md
- Client-side Blob download — zero server round-trip
- Export button disabled during streaming and when messages empty

### Phase 25: Motion Animations

**Goal**: New messages and UI transitions have fluid entrance animations consistent with the Microsoft Copilot aesthetic, with full prefers-reduced-motion compliance.
**Depends on**: Phases 22, 23, 24 (component structure must be settled before applying motion wrappers)
**Plans**: 2 plans

Plans:
- [x] 25-01: motion package install, LazyMotion + MotionConfig providers, CSS cleanup
- [x] 25-02: m.div entrance animations on messages with streaming guard, historical message gate, feedback button scale class wiring

**Details:**
- motion@12.38.0 installed (React 19 compatible, confirmed via spike)
- LazyMotion + domAnimation for tree-shaken bundle; MotionConfig reducedMotion="user" at root
- Provider order: MotionConfig > LazyMotion > FluentProvider
- AssistantMessage: 200ms ease-out fade+slide (isNew && !isStreaming guard)
- UserMessage: 150ms ease-out fade+slide (isNew guard)
- loadedCountRef gate: snapshot messages.length on thread switch for historical message detection
- Sidebar transition tuned to 225ms ease-in-out (CSS only)
- feedback-scale-btn CSS :active scale(0.88) with prefers-reduced-motion override

## Milestone Summary

**Decimal Phases:**
- None — all phases sequential (21-25)

**Key Decisions:**
- App Roles over groupMembershipClaims for access gating (no overage at 80K+ users)
- role_required is the canonical route decorator (login_required retained but unused)
- AuthStatus discriminated union replaces loading boolean
- AccessDenied renders before ThreadProvider/ChatProvider to prevent cascading 403s
- SSE 401/403 triggers page reload so AuthGuard re-evaluates
- migrate_db() runs on every startup — idempotent DDL for feedback table and FTS5
- POST vote=null retracts (one endpoint for set and clear)
- Comment truncated 500 chars at API layer
- FTS5 unicode61 tokenizer only (porter over-stems Exchange terms)
- INSERT OR IGNORE backfill for FTS idempotency
- _build_fts5_query quotes each token to neutralise operators
- Tool panels before assistant content in Markdown export (mirrors UI order)
- motion@12.38.0 (not framer-motion) with LazyMotion + domAnimation
- Wrapper pattern: const Wrapper = condition ? m.div : 'div' for conditional animation
- loadedCountRef gate for historical vs new message distinction

**Issues Resolved:**
- React 19 + motion compatibility confirmed (was MEDIUM confidence blocker)
- ThumbLike16 icons not available from main @fluentui/react-icons; standard-sized variants used instead
- Ctrl+K deferred focus via setTimeout(0) when expanding collapsed sidebar

**Issues Deferred:**
- Atlas.User App Role must be created in Entra admin center (admin dependency)
- CHATGPT_ENDPOINT not in AWS Secrets Manager pipeline

**Technical Debt Incurred:**
- Sidebar CSS transition lacks prefers-reduced-motion override (low severity CSS polish)
- login_required returns 302 not 401 for API routes (mitigated by role_required)
- Historical tool panels always show "Done" badge
- Historical tool panels lose elapsed time
- AuthContext.error field unused
