Modify `InputArea.tsx` with these specific changes:

1. **Textarea `readOnly` during streaming**: Replace `disabled={disabled}` on the textarea with `readOnly={isStreaming}` and `disabled={disabled && !isStreaming}`. The `readOnly` attribute preserves visual state while preventing input. The `disabled` prop from AppLayout (`!activeThreadId`) should still disable when no thread is active AND not streaming.

   Actually, simpler: the textarea should be `readOnly` when `isStreaming` is true, and `disabled` when the `disabled` prop is true and NOT streaming. This prevents the textarea from being both disabled and readOnly simultaneously.

   ```tsx
   <textarea
     ref={textareaRef}
     className="chat-input"
     placeholder="Ask Atlas anything about Exchange..."
     rows={1}
     value={message}
     onChange={handleChange}
     onKeyDown={handleKeyDown}
     readOnly={isStreaming}
     disabled={disabled && !isStreaming}
   />
   ```

2. **Stop button** — Replace the text "Stop" button with a styled circular button. Change the stop branch from:
   ```tsx
   <button className="send-btn" onClick={onCancel} disabled={isButtonDisabled}>
     Stop
   </button>
   ```
   to:
   ```tsx
   <button
     className="stop-btn"
     onClick={onCancel}
     aria-label="Stop generating"
     type="button"
   />
   ```
   The stop button is never disabled — user can always cancel. Remove the `isButtonDisabled` check for the stop path. The `::before` pseudo-element provides the white square icon via CSS.

3. **Send button** — keep as-is but ensure it has `type="button"` to prevent accidental form submission:
   ```tsx

   </button>
   ```

   Remove the `!isStreaming &&` from the condition since `isButtonDisabled` is only used on the Send button branch (which only renders when NOT streaming).

5. **Placeholder text** — change from `"Type a message..."` to `"Ask Atlas anything about Exchange..."`.

Do NOT change the `handleSubmit`, `handleChange`, `handleKeyDown`, or `adjustHeight` functions — they work correctly.
