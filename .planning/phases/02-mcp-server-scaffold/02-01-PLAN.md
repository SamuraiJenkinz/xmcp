Create exchange_mcp/server.py with the following structure, in this exact order:

1. **Logging configuration as the absolute first executable lines** (before any other imports):
   ```python
   import logging
   import sys

   logging.basicConfig(
       level=logging.INFO,
       format="%(asctime)s %(levelname)-8s %(name)s: %(message)s",
       stream=sys.stderr,
   )
   logger = logging.getLogger("exchange_mcp.server")
   ```

2. **Remaining imports** (after logging is configured):
   - `import json`, `import signal`, `import time`
   - `import anyio`
   - `import mcp.types as types`
   - `from mcp.server import Server`
   - `from mcp.server.stdio import stdio_server`
   - `from exchange_mcp.exchange_client import ExchangeClient`

3. **Server instance**: `server = Server("exchange-mcp", version="0.1.0")`

4. **Module-level ExchangeClient reference**: `_exchange_client: ExchangeClient | None = None` (set during startup, used by call_tool handler later)

5. **list_tools handler** using `@server.list_tools()` decorator:
   - Return a list containing one Tool: name="ping", description="Test server connectivity. Use when checking if the Exchange MCP server is running and responsive.", inputSchema={"type": "object", "properties": {}, "required": []}
   - This placeholder will be replaced with all 15 tools in Plan 02

6. **call_tool handler** using `@server.call_tool()` decorator:
   - Receives (name: str, arguments: dict | None)
   - Returns list[types.TextContent]
   - Log tool_call with name and arguments at start
   - For name=="ping": return [types.TextContent(type="text", text='"pong"')]
   - For unknown tool name: raise ValueError(f"Unknown tool: {name}")
   - Wrap the entire body in try/except that logs tool_ok or tool_error with duration
   - On exception: log the error, then `raise RuntimeError(sanitized_message) from None` so the SDK creates isError=True with a clean message (not raw PS traceback)
   - Use time.monotonic() for duration measurement

7. **SIGTERM handler**: Register `signal.signal(signal.SIGTERM, handler)` that logs "SIGTERM received -- shutting down" and calls `sys.exit(0)`. Wrap in try/except for Windows compatibility (SIGTERM may behave differently).

8. **async def main()** function:
   - Import and instantiate ExchangeClient
   - Assign to module-level `_exchange_client` via `global _exchange_client`
   - Call `await client.verify_connection()`
   - If False: `logger.error("FATAL: Cannot reach Exchange Online. Check credentials (auth_mode=%s) and network connectivity. Server will not start.", client.auth_mode)` then `sys.exit(1)`
   - Count tools via `await handle_list_tools()`
   - Log startup banner: `"exchange-mcp v0.1.0 started | auth=%s | tools=%d | endpoint=Exchange Online"`
   - Open stdio transport: `async with stdio_server() as (read_stream, write_stream):`
   - Run server: `await server.run(read_stream, write_stream, server.create_initialization_options())`

9. **Entry point**: `if __name__ == "__main__": anyio.run(main)`

Important constraints:
- NEVER use print() anywhere -- all output via logger (which goes to stderr)
- The call_tool handler should use a helper function `_sanitize_error(exc)` that strips "stderr:" sections and "PowerShell exited with code" prefixes from error messages. Include transient error detection to append "This is usually transient -- try again in a moment." hint. Define the transient/non-transient pattern lists as module constants. See RESEARCH.md Pattern 3 for the exact implementation.
- The startup validation MUST run BEFORE `async with stdio_server()` -- not inside it
