  <files>chat_app/auth.py</files>
  <action>
Add `import datetime` at the top of auth.py.

Add a module-level constant:
```python
REQUIRED_ROLE = "Atlas.User"
```

Create a new `role_required` decorator below the existing `login_required`. The decorator:

1. Checks `session.get("user")` — if falsy:
   - For paths starting with `/api/` or `/chat/`: return `jsonify({"error": "authentication_required", "message": "Login required"}), 401`
   - Otherwise: `redirect(url_for("catch_all", path=""))`

2. If user exists, check `REQUIRED_ROLE not in user.get("roles", [])` — if not in roles:
   - Log a warning: `logger.warning("403 Forbidden: upn=%s endpoint=%s ts=%s", upn, request.path, datetime.datetime.utcnow().isoformat())`
   - For paths starting with `/api/` or `/chat/`: return `jsonify({"error": "forbidden", "message": "Atlas.User role required", "required_role": REQUIRED_ROLE, "upn": user.get("preferred_username", "")}), 403`
   - Otherwise: `redirect(url_for("catch_all", path=""))`

3. If role check passes: call `f(*args, **kwargs)`

Keep `login_required` in the file (it's still a valid function) but it will no longer be used as a route decorator.

Export `role_required` alongside `login_required` in the module (both importable, but only `role_required` used on routes going forward).
  </action>
