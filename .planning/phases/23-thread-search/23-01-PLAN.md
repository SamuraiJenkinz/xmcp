  <name>Task 1: FTS5 virtual table, triggers, and backfill in schema + migrate_db</name>
  <files>chat_app/schema.sql, chat_app/db.py</files>
  <action>
1. In schema.sql, add after the existing table definitions:

   a) `CREATE VIRTUAL TABLE IF NOT EXISTS threads_fts USING fts5(body, tokenize='unicode61');`

   b) Three triggers using json_each() to sync FTS content:
      - `messages_fts_ai` AFTER INSERT ON messages: DELETE then INSERT into threads_fts, extracting only user+assistant role content via json_extract, group_concat with space separator.
      - `messages_fts_au` AFTER UPDATE ON messages: Same DELETE+INSERT pattern for updated conversations.
      - `threads_fts_ad` AFTER DELETE ON threads: DELETE from threads_fts WHERE rowid = OLD.id.

   c) Backfill query: `INSERT OR IGNORE INTO threads_fts(rowid, body) SELECT m.thread_id, group_concat(...) FROM messages m, json_each(m.messages_json) j WHERE role IN ('user','assistant') AND content IS NOT NULL GROUP BY m.thread_id;`

   All use `CREATE ... IF NOT EXISTS` / `CREATE TRIGGER IF NOT EXISTS` for idempotency.

2. In db.py, extend migrate_db() to execute the FTS DDL statements. Follow the existing pattern of additive schema migration using executescript or individual execute calls. The FTS table creation, trigger creation, and backfill INSERT OR IGNORE are all idempotent and safe to run on every startup.

   IMPORTANT: The existing pattern in db.py uses executescript for DDL. FTS5 virtual table creation and trigger creation work fine with executescript. The backfill INSERT OR IGNORE also works. Add these as a new block in migrate_db() after the existing migration statements.
  </action>

<task type="auto">
  <name>Task 2: Search endpoint with query builder and user scoping</name>
  <files>chat_app/conversations.py</files>
  <action>
1. Add a `_build_fts5_query(raw: str) -> str | None` helper function in conversations.py (private, not exported). This function:
   - Strips and splits the raw input on whitespace
   - Returns None for empty/blank input
   - Wraps each token in double quotes (escaping any literal double quotes) with trailing * for prefix matching
   - Example: 'DAG health' -> '"DAG"* "health"*'

2. Add a `_strip_mark_tags(snippet: str) -> str` helper that removes `<mark>` and `</mark>` tags from snippet() output, keeping inner text. Use `re.sub(r'</?mark>', '', snippet)`. Import `re` at the top.

3. Add the search endpoint on the existing `conversations_bp` Blueprint:
   ```
   @conversations_bp.route("/api/threads/search", methods=["GET"])
   @role_required
   def search_threads():
   ```
   - Read `q` from request.args, strip it
   - Call _build_fts5_query(q); if None, return jsonify([])
   - Execute FTS5 query with JOIN threads for user scoping:
     SELECT t.id, t.name, t.updated_at, snippet(threads_fts, 0, '<mark>', '</mark>', '...', 16) AS snippet
     FROM threads_fts JOIN threads t ON threads_fts.rowid = t.id
     WHERE threads_fts MATCH ? AND t.user_id = ?
     ORDER BY rank LIMIT 20
   - Wrap the execute in try/except: on any Exception, return jsonify([])
   - Strip mark tags from each row's snippet before returning
   - Return jsonify([dict with id, name, updated_at, snippet for each row])
   - Use _user_id() helper (already exists in conversations.py) for user scoping

   IMPORTANT: The @role_required decorator must come before the endpoint. This ensures 401/403 for unauthenticated/unauthorized users — consistent with all other endpoints in this blueprint.
  </action>
