**MessageList.tsx — historical message gate:**

The goal: Only messages added AFTER the current thread was loaded/switched should animate. Historical messages (loaded via SET_MESSAGES on thread switch) should NOT animate.

Implementation approach — track message count at last thread load:

1. Add `useRef` (already imported) to track the message count at the time of last thread load:
   ```tsx
   const loadedCountRef = useRef(messages.length);
   ```

2. When `activeThreadId` changes, reset the loaded count to the current messages length. Use a `useEffect`:
   ```tsx
   useEffect(() => {
     // When thread changes, snapshot current message count as "historical"
     // This runs after SET_MESSAGES populates the messages array
     loadedCountRef.current = messages.length;
   }, [activeThreadId]);
   ```

   Important: This effect fires when activeThreadId changes. At that point, messages has already been set by the thread-switch handler (SET_MESSAGES). So `messages.length` reflects the loaded historical count. Any messages added after this (via ADD_USER_MESSAGE or FINALIZE_STREAMING) will have index >= loadedCountRef.current, marking them as "new".

3. In the `messages.map()`, compute `isNew` for each message:
   ```tsx
   {messages.map((msg, idx) => {
     const isNew = idx >= loadedCountRef.current;
     if (msg.type === 'user') {
       return <UserMessage key={idx} content={msg.content} timestamp={msg.timestamp} isNew={isNew} />;
     }
     const assistantIdx = messages
       .slice(0, idx + 1)
       .filter(m => m.type === 'assistant').length - 1;
     return (
       <AssistantMessage
         key={idx}
         content={msg.content}
         toolPanels={msg.toolPanels}
         timestamp={msg.timestamp}
         threadId={activeThreadId ?? undefined}
         messageIndex={assistantIdx}
         isNew={isNew}
       />
     );
   })}
   ```

4. The streaming AssistantMessage does NOT need `isNew` — it already passes `isStreaming={true}` which takes priority in AssistantMessage's rendering logic.

**Edge case — new thread (no history):** When a new thread is created, `activeThreadId` changes and `messages` is `[]`, so `loadedCountRef.current = 0`. The first user message (idx 0) has `0 >= 0 = true` so it animates. Correct.

**Edge case — initial load with no thread:** `activeThreadId` starts as `null`, `messages` starts as `[]`. `loadedCountRef.current = 0`. First message animates. Correct.
