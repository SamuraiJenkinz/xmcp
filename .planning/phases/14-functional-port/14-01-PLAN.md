```typescript
// Thread from GET /api/threads
export interface Thread {
  id: number;
  name: string;
  updated_at: string;
}

// Raw message from GET /api/threads/:id/messages (OpenAI format)
export interface RawMessage {
  role: 'system' | 'user' | 'assistant' | 'tool';
  content: string | null;
  tool_calls?: Array<{
    id: string;
    type: 'function';
    function: { name: string; arguments: string };
  }>;
  tool_call_id?: string;
  name?: string;
}

// Parsed tool panel data for rendering
export interface ToolPanelData {
  name: string;
  params: Record<string, unknown>;
  result: string | null;
  status: 'success' | 'error';
}

// Display-ready message after parsing raw messages
export interface DisplayMessage {
  type: 'user' | 'assistant';
  content: string;
  toolPanels?: ToolPanelData[];
}

// SSE event types from POST /chat/stream
export type SSEEvent =
  | { type: 'text'; delta: string }
  | { type: 'tool'; name: string; status: 'success' | 'error'; params: Record<string, unknown>; result: string | null }
  | { type: 'thread_named'; thread_id: number; name: string }
  | { type: 'done' }
  | { type: 'error'; message: string };

// Streaming message state (separate from messages array)
export interface StreamingMessageState {
  content: string;
  toolPanels: ToolPanelData[];
}

// User session from GET /api/me
export interface User {
  displayName: string;
  email: string;
  oid: string;
}
```

**frontend/src/contexts/ThreadContext.tsx:**
- Create ThreadContext with `useReducer`
- State shape: `{ threads: Thread[]; activeThreadId: number | null }`
- Action types (match research exactly):
  - `SET_THREADS` — replace entire threads array (initial load, after delete)
  - `ADD_THREAD` — prepend a new thread
  - `REMOVE_THREAD` — filter out by threadId
  - `RENAME_THREAD` — update name for a specific threadId
  - `SET_ACTIVE` — change activeThreadId
  - `BUMP_THREAD` — move a thread to top of list (after `done` event, re-sort by updated_at)
- ThreadProvider: on mount, call `listThreads()` and dispatch `SET_THREADS`
- Export `useThreads()` hook and `ThreadProvider` component
- The hook should return `{ threads, activeThreadId, dispatch }` — dispatch is typed with ThreadAction

**frontend/src/contexts/ChatContext.tsx:**
- Create ChatContext with `useReducer`
- State shape: `{ messages: DisplayMessage[]; streamingMessage: StreamingMessageState | null; isStreaming: boolean; error: string | null }`
- **Critical:** `streamingMessage` is separate from `messages[]` array (see research anti-patterns)
- Action types:
  - `SET_MESSAGES` — replace messages array (thread switch / history load). Does NOT touch streamingMessage.
  - `APPEND_STREAMING_CHUNK` — append delta to streamingMessage.content. If streamingMessage is null, create it first with empty content and empty toolPanels.
  - `ADD_TOOL_EVENT` — push a ToolPanelData into streamingMessage.toolPanels
  - `FINALIZE_STREAMING` — move streamingMessage into messages[] as a DisplayMessage, clear streamingMessage, set isStreaming=false
  - `CANCEL_STREAMING` — if streamingMessage exists, append "[response cancelled]" to content, then finalize it into messages[]. Set isStreaming=false.
  - `SET_STREAMING` — set isStreaming flag
  - `SET_ERROR` — set error string
  - `ADD_USER_MESSAGE` — append a `{ type: 'user', content }` DisplayMessage to messages[]. Used by AppLayout before starting a stream so the user message appears immediately.
  - `CLEAR_ERROR` — clear error
- Export `useChat()` hook and `ChatProvider` component
- The hook returns `{ messages, streamingMessage, isStreaming, error, dispatch }`
