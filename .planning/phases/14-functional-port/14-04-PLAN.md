**frontend/src/components/ChatPane/MarkdownRenderer.tsx:**

**frontend/src/components/shared/CopyButton.tsx:**
Port directly from research code example. This fixes DEBT-02.

- Props: `{ getText: () => string }` — lazy evaluation at click time (not a static string)
- Use `useState` for `copied` boolean
- `handleCopy`: call `navigator.clipboard.writeText(getText())`, set copied=true, setTimeout 1500ms to reset (matches app.js timing exactly)
- Catch clipboard errors and console.error
- Render a `<button>` with className `copy-btn`, text "Copied!" or "Copy"
- Use `useCallback` for handleCopy with `getText` in deps

**DEBT-02 fix explanation:** By accepting `getText` as a function called at click time, CopyButton works identically on streaming messages (where content is still accumulating) and historical messages (loaded from DB). The old vanilla JS approach relied on a specific DOM element reference that didn't exist for historical messages.

- Render native `<details className="tool-panel">` with `<summary className="tool-panel-summary">`

- Use the existing CSS class names from app.js: `tool-panel`, `tool-panel-summary`, `tool-panel-icon`, `tool-panel-name`, `tool-panel-status`, `tool-panel-body`, `tool-panel-label`, `tool-panel-json`, `tool-panel-result`
