Create `frontend/src/components/Sidebar/ThreadList.tsx`:

**Behavior — port from app.js sidebar management:**
- Reads threads and activeThreadId from `useThreads()` context hook
- Reads `isStreaming` from `useChat()` context hook (needed to abort stream on switch)
- Renders a "New Chat" button at top of sidebar
- Maps `threads` array to `<ThreadItem>` components

**New Chat button handler:**
1. Call `createThread()` API
2. Dispatch `ADD_THREAD` with the returned thread
3. Dispatch `SET_ACTIVE` with the new thread ID
4. Dispatch `SET_MESSAGES` to ChatContext with empty array (clear chat pane)

**Thread select handler (critical — abort stream on switch):**
1. If `isStreaming` is true, the parent must cancel the stream first — accept a `cancelStream` prop or use a callback pattern
2. Dispatch `SET_ACTIVE` with new thread ID
3. Dispatch `SET_MESSAGES` to ChatContext with empty array (clear immediately for responsiveness)
4. Call `getMessages(threadId)` API
5. Parse the raw messages through `parseHistoricalMessages()` from `../utils/parseHistoricalMessages`
6. Dispatch `SET_MESSAGES` with the parsed DisplayMessage array

**Thread rename handler:**
1. Call `renameThread(threadId, newName)` API
2. Dispatch `RENAME_THREAD` to ThreadContext

**Thread delete handler:**
1. Call `deleteThread(threadId)` API
2. Dispatch `REMOVE_THREAD` to ThreadContext
3. If deleted thread was active:
   - If other threads remain, set the first one as active and load its messages
   - If no threads remain, clear activeThreadId and messages

**Props interface:**
```typescript
interface ThreadListProps {
  onCancelStream?: () => void;  // Called before thread switch if streaming
}
```

The component needs access to both `useThreads()` and `useChat()` dispatch. Import `parseHistoricalMessages` and `getMessages` for history loading.
