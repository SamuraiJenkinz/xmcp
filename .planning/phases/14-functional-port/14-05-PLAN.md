  <action>
Create `frontend/src/components/ChatPane/InputArea.tsx`. This is a direct port of the textarea handling from app.js.

**Props:**
```typescript
interface InputAreaProps {
  onSend: (message: string) => void;
  onCancel: () => void;
  isStreaming: boolean;
  disabled?: boolean;  // true while no thread is active
}
```

**Auto-resize textarea (port from app.js resizeInput):**
- `textareaRef = useRef<HTMLTextAreaElement>(null)`
- `adjustHeight()`: set height to 'auto', then `Math.min(scrollHeight, 200)` px — 200px max matches app.js exactly
- `resetHeight()`: set height to 'auto'
- Call `adjustHeight()` in onChange handler
- Call `resetHeight()` after submit

**Message state:** Use `useState<string>('')` for the input value. Controlled textarea.

**Keyboard shortcuts (match app.js exactly):**
- **Enter** (no shift): Submit the message (call `handleSubmit()`). Prevent default (no newline).
- **Shift+Enter**: Insert newline (default behavior, do nothing special).
- **Escape**: If streaming, call `onCancel()`. If not streaming, optionally clear input.
- Handle in `onKeyDown` handler.

**Submit handler:**
1. Trim the message. If empty, return early.
2. Call `onSend(message.trim())`
3. Clear input state to ''
4. Call `resetHeight()`
5. Re-focus the textarea

**Send/Stop button:**
- If `isStreaming`: show "Stop" button that calls `onCancel()`
- If not streaming: show "Send" button that calls `handleSubmit()`
- Button disabled if: not streaming AND input is empty, OR `disabled` prop is true

**Render:**
- Container div with className `input-area`
- `<textarea>` with ref, className `chat-input`, placeholder "Type a message...", rows={1}
- Button with className `send-btn`
- Auto-focus textarea on mount (useEffect + ref.focus())

**frontend/src/components/AppLayout.tsx:**
- This is the main layout component that combines sidebar and chat pane
- Import: ThreadList, MessageList, InputArea, Header
- Import: useThreads, useChat hooks
- Import: useStreamingMessage hook
- **Wire useStreamingMessage:**
  - Create the streaming message hook with callbacks that dispatch to ChatContext:
    - `onText`: dispatch `APPEND_STREAMING_CHUNK`
    - `onTool`: dispatch `ADD_TOOL_EVENT`
    - `onThreadNamed`: dispatch `RENAME_THREAD` to ThreadContext + `BUMP_THREAD` to move thread to top
    - `onDone`: dispatch `FINALIZE_STREAMING`
    - `onError`: dispatch `SET_ERROR`
    - `onCancel`: dispatch `CANCEL_STREAMING`
  - Before calling startStream: dispatch `SET_STREAMING` with true, and dispatch `ADD_USER_MESSAGE` with the user's message text (appends to messages[] optimistically)
- **handleSend function:**
  1. If no activeThreadId: create a new thread first, then proceed
  2. Add user message to display: dispatch `ADD_USER_MESSAGE` with `message` (ChatContext appends `{ type: 'user', content: message }` to messages[])
  3. Call `startStream(message, activeThreadId)`
- **handleCancel:** call `cancelStream()` from the hook
- **Layout structure:**
  ```html

    <main className="chat-pane">
      <Header theme={theme} onToggleTheme={onToggleTheme} />
      <MessageList />
      <InputArea
        onSend={handleSend}
        onCancel={handleCancel}
        isStreaming={isStreaming}

function AuthGuard({ children }: { children: React.ReactNode }) {
  const { user, loading } = useAuth();
  if (loading) return <div className="loading">Loading...</div>;
  if (!user) {
    window.location.href = '/login';
    return null;
  }
  return <>{children}</>;
}
```
