  <action>
    1. Run `npm create vite@latest frontend -- --template react-ts` from the project root (`C:\xmcp`). This generates the standard Vite + React + TypeScript scaffold.

    2. `cd frontend` and install the additional dependencies:
       ```
       npm install @fluentui/react-components
       npm install -D tailwindcss @tailwindcss/vite
       ```

    3. Edit `frontend/vite.config.ts` to match this exact configuration:
       ```typescript
       import { defineConfig } from 'vite'
       import react from '@vitejs/plugin-react'
       import tailwindcss from '@tailwindcss/vite'

       export default defineConfig({
         plugins: [react(), tailwindcss()],
         build: {
           outDir: '../frontend_dist',
           emptyOutDir: true,
         },
         server: {
           port: 5173,
           proxy: {
             '/api': { target: 'http://127.0.0.1:5000', changeOrigin: true },
             '/login': { target: 'http://127.0.0.1:5000', changeOrigin: true },
             '/logout': { target: 'http://127.0.0.1:5000', changeOrigin: true },
             '/auth': { target: 'http://127.0.0.1:5000', changeOrigin: true },
           },
         },
       })
       ```

    4. Edit `frontend/index.html`: change `<div id="root">` to `<div id="app">` to match the Flask convention. Update the title to "Atlas".

    5. Delete the Vite template boilerplate files that we don't need: `src/App.css`, `src/assets/` directory. Keep `src/vite-env.d.ts`.

    6. Create empty directories: `frontend/src/components/`, `frontend/src/hooks/`, `frontend/src/api/` (add `.gitkeep` files so git tracks them).

    IMPORTANT: Do NOT set `base: './'` in vite.config — leave it as default `'/'`. Setting it breaks absolute asset paths when Flask serves the SPA.
  </action>
  <verify>
    - `ls frontend/package.json` confirms the project exists
    - `cd frontend && npm ls @fluentui/react-components` shows it installed
    - `cd frontend && npm ls tailwindcss` shows it installed
    - `grep '"app"' frontend/index.html` confirms the mount point ID is "app"
    - `grep 'outDir' frontend/vite.config.ts` shows `'../frontend_dist'`
    - `grep 'proxy' frontend/vite.config.ts` shows proxy configuration exists
  </verify>
  <done>
    frontend/ directory exists with all dependencies installed; vite.config.ts has proxy to Flask :5000 and build output to ../frontend_dist; index.html mounts on div#app; empty component/hook/api directories created
  </done>

<task type="auto">
  <name>Task 2: Create App.tsx, main.tsx, index.css with Fluent UI + Tailwind and verify build</name>
  <files>
    frontend/src/main.tsx
    frontend/src/App.tsx
    frontend/src/index.css
  </files>
  <action>
    1. Replace `frontend/src/index.css` with exactly:
       ```css
       @import "tailwindcss" prefix(tw);
       ```
       This is the Tailwind v4 CSS-native prefix configuration. Classes will be `tw:flex`, `tw:min-h-screen`, etc. (colon syntax, NOT hyphen — this is a v4 breaking change from v3).

    2. Replace `frontend/src/main.tsx` with:
       ```typescript
       import { StrictMode } from 'react'
       import { createRoot } from 'react-dom/client'
       import './index.css'
       import App from './App'

       createRoot(document.getElementById('app')!).render(
         <StrictMode>
           <App />
         </StrictMode>
       )
       ```
       IMPORTANT: Mount on `'app'` not `'root'` — must match the div ID in index.html.

    3. Replace `frontend/src/App.tsx` with:
       ```typescript
       import { FluentProvider, webDarkTheme, Title1 } from '@fluentui/react-components'

       export default function App() {
         return (
           <FluentProvider theme={webDarkTheme}>
             <div className="tw:flex tw:flex-col tw:min-h-screen tw:items-center tw:justify-center">
               <Title1>Atlas</Title1>
               <p className="tw:mt-4">React shell is running. Fluent UI and Tailwind are configured.</p>
             </div>
           </FluentProvider>
         )
       }
       ```
       This serves as a smoke test: FluentProvider with webDarkTheme proves Fluent works, Title1 proves component imports work, tw: classes prove Tailwind prefix works.

    4. Run `cd frontend && npm run build` to produce the production bundle in `frontend_dist/` at the project root.

    5. Verify the build output:
       - `frontend_dist/index.html` exists
       - `frontend_dist/assets/` contains .js and .css files
       - The built index.html contains `<div id="app">`

    6. Add `frontend/node_modules/` and `frontend_dist/` to the project's `.gitignore` if not already present. Also add `frontend/.vite/` (Vite cache).
  </action>
  <verify>
    - `cd frontend && npm run build` completes with exit code 0
    - `ls frontend_dist/index.html` confirms the build output exists at the project root level
    - `ls frontend_dist/assets/` shows .js and .css files
    - `grep 'app' frontend_dist/index.html` confirms the mount point survived the build
    - `grep 'node_modules' .gitignore` confirms gitignore is updated
  </verify>

  </done>
