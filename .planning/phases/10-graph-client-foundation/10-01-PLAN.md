  <name>Task 1: Add Graph constants to config.py and create graph_client.py skeleton</name>
  <files>chat_app/config.py, chat_app/graph_client.py</files>
  <action>
1. In `chat_app/config.py`, add three class attributes to `Config` after the Azure OpenAI section:
   ```python
   # Microsoft Graph API
   GRAPH_BASE_URL: str = "https://graph.microsoft.com/v1.0"
   GRAPH_SEARCH_MAX_RESULTS: int = 25
   GRAPH_TIMEOUT: int = 10
   ```
   These are constants, NOT env vars. No changes to `update_from_secrets()` needed.

2. Create `chat_app/graph_client.py` with:
   - Module docstring: `"""Microsoft Graph API client — isolated singleton for colleague lookup."""`
   - Imports: `from __future__ import annotations`, `base64`, `json`, `logging`, `time`, `msal`, `requests`
   - Module-level logger: `logger = logging.getLogger(__name__)`
   - Module-level state: `_cca: msal.ConfidentialClientApplication | None = None`, `_graph_enabled: bool = False`
   - `_GRAPH_SCOPES = ["https://graph.microsoft.com/.default"]`
   - `init_graph(client_id: str, client_secret: str, tenant_id: str) -> None` function that:
     a. Guards with `if not all([client_id, client_secret, tenant_id])` — log warning listing which vars are missing and return
     b. Creates `_cca = msal.ConfidentialClientApplication(client_id=client_id, client_credential=client_secret, authority=f"https://login.microsoftonline.com/{tenant_id}")`
     c. Calls `_cca.acquire_token_for_client(scopes=_GRAPH_SCOPES)`
     d. If `"error"` in result: log error with the exact admin consent URL `https://login.microsoftonline.com/{tenant_id}/adminconsent?client_id={client_id}` and return without setting `_graph_enabled = True`
     e. Calls `_verify_roles(result["access_token"], tenant_id, client_id)`
     f. Sets `_graph_enabled = True` and logs info "Graph client initialized successfully"
   - `_verify_roles(access_token: str, tenant_id: str, client_id: str) -> None` function that:
     a. Splits JWT on ".", takes index 1 (payload)
     b. Adds base64 padding: `padded = payload_b64 + "=" * (4 - len(payload_b64) % 4)`
     c. Decodes with `base64.urlsafe_b64decode(padded)` and `json.loads`
     d. Checks `roles` claim for `User.Read.All` and `ProfilePhoto.Read.All`
     e. If any missing: logs ERROR with missing roles list and the admin consent URL
     f. Wrapped in try/except that logs warning on decode failure
   - `is_graph_enabled() -> bool` convenience function returning `_graph_enabled`

Do NOT add the `init_graph()` call to `app.py` yet — that happens in Plan 02 after token acquisition logic is complete.
