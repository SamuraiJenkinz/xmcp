  <files>chat_app/graph_client.py</files>
  <action>
Add the following functions to graph_client.py (the module skeleton with init_graph, _verify_roles, and is_graph_enabled already exists from Plan 01):

1. `_get_token() -> str | None`:
   - If `_cca is None`, return `None`
   - Call `_cca.acquire_token_for_client(scopes=_GRAPH_SCOPES)`
   - If `"access_token"` in result, return it. MSAL handles cache automatically — calling this on every request is correct and cheap on cache hits (verified: MSAL 1.34.0 returns cached token if >5 min remaining).
   - If error, log ERROR with `result.get("error")` and `result.get("error_description")`, return `None`
   - IMPORTANT: Check for `"access_token" in result`, NOT `if result` — MSAL always returns a dict, errors are truthy dicts.

2. `_make_headers(*, search: bool = False) -> dict[str, str] | None`:
   - Call `_get_token()`. If None, return None.
   - Build dict: `{"Authorization": f"Bearer {token}"}`
   - If `search=True`, add `"ConsistencyLevel": "eventual"` — this is MANDATORY for $search on directory objects, without it Graph returns 400.
   - Return the headers dict.

3. `_graph_request_with_retry(method: str, url: str, *, headers: dict, params: dict | None = None, max_retries: int = 3, timeout: int = 10) -> requests.Response`:
   - Loop `max_retries` times (use `for attempt in range(max_retries)`)
   - Call `requests.request(method, url, headers=headers, params=params, timeout=timeout)`
   - If status code is 429 or 503: read `Retry-After` header (default to `2 ** attempt` if absent), log warning with status code and attempt number, `time.sleep(retry_after)`, continue
   - For any other status code: return the response immediately (caller handles 404, raise_for_status, etc.)
   - On `requests.exceptions.Timeout`: log warning, `time.sleep(2 ** attempt)`, continue
   - After all retries exhausted: raise the last exception, or `RuntimeError("Graph request failed after retries")` if no exception was caught

Import `from chat_app.config import Config` and use `Config.GRAPH_TIMEOUT` as the default timeout value in `_graph_request_with_retry` instead of hardcoded 10.
