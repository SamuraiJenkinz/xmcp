  <name>Task 1: Implement search_users() with $search and ConsistencyLevel header</name>
  <files>chat_app/graph_client.py</files>
  <action>
Add `search_users(term: str) -> list[dict]` to graph_client.py:

1. Early return: if `not _graph_enabled` or `not term or not term.strip()`, return `[]`
2. Sanitize the search term: `term = term.strip()`. Do NOT quote-escape the term — Graph's $search syntax handles quotes internally.
3. Build the request:
   - URL: `f"{Config.GRAPH_BASE_URL}/users"`
   - Headers: `_make_headers(search=True)` — this adds BOTH the Bearer token AND `ConsistencyLevel: eventual`. If returns None, return `[]`.
   - Params dict:
     ```python
     {
         "$search": f'"displayName:{term}" OR "mail:{term}"',
         "$select": "id,displayName,mail,jobTitle,department",
         "$filter": "accountEnabled eq true",
         "$top": str(Config.GRAPH_SEARCH_MAX_RESULTS),
     }
     ```
4. Call `_graph_request_with_retry("GET", url, headers=headers, params=params)`
5. Call `resp.raise_for_status()` — let HTTP errors propagate (429/503 already handled by retry helper)
6. Return `resp.json().get("value", [])`
7. Wrap the entire request+parse in try/except. On any exception: log ERROR with the exception, return `[]`. Graph search failure should never crash the caller.

CRITICAL: The `ConsistencyLevel: eventual` header MUST be on every search request. Without it, Graph returns 400. This is enforced by using `_make_headers(search=True)` — never construct headers manually for search.

CRITICAL: The `$search` value must use the exact format `"displayName:{term}" OR "mail:{term}"` with the double-quotes inside the string value. These quotes are part of the OData $search syntax, not Python string delimiters.
  </action>
