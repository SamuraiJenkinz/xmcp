    Create new file `exchange_mcp/feedback_analytics.py`. This module isolates feedback analytics from Exchange tool handlers (requirement FBAN-08).

    The module must contain:

    1. **Module docstring** explaining this is for read-only feedback analytics via SQLite.

    2. **Imports**: `sqlite3`, `os`, `asyncio`, `datetime` (datetime, timezone, timedelta, date as date_type), `typing` (Any), `urllib.parse` (quote). Do NOT import anything from `chat_app` — no Flask dependencies.

    3. **SQL constants** (module-level):
       - `SUMMARY_SQL`: SELECT COUNT(*) AS total_votes, SUM(CASE WHEN vote='up' THEN 1 ELSE 0 END) AS up_votes, SUM(CASE WHEN vote='down' THEN 1 ELSE 0 END) AS down_votes, SUM(CASE WHEN comment IS NOT NULL AND comment != '' THEN 1 ELSE 0 END) AS comment_count FROM feedback WHERE created_at >= ? AND created_at <= ?
       - `TREND_SQL`: SELECT substr(created_at, 1, 10) AS day, COUNT(*) AS total, SUM(CASE WHEN vote='up' THEN 1 ELSE 0 END) AS up_votes, SUM(CASE WHEN vote='down' THEN 1 ELSE 0 END) AS down_votes FROM feedback WHERE created_at >= ? AND created_at <= ? GROUP BY day ORDER BY day ASC
       - `LOW_RATED_SQL`: SELECT f.created_at, t.name AS thread_name, f.comment, CASE WHEN f.comment IS NOT NULL AND f.comment != '' THEN 1 ELSE 0 END AS has_comment, f.thread_id, f.assistant_message_idx FROM feedback f JOIN threads t ON t.id = f.thread_id WHERE f.vote = 'down' AND f.created_at >= ? AND f.created_at <= ? ORDER BY f.created_at DESC LIMIT ?

    4. **`_open_ro(db_path: str) -> sqlite3.Connection`** helper:
       - URL-encode db_path for Windows spaces safety: `from urllib.parse import quote; uri = f"file:{quote(db_path)}?mode=ro"`
       - `conn = sqlite3.connect(uri, uri=True)`
       - `conn.row_factory = sqlite3.Row`
       - Return conn
       - Do NOT set any PRAGMAs (no journal_mode=WAL — the file already has it, and read-only connections cannot write PRAGMAs)

    5. **`_zero_fill_trend(db_rows: dict[str, dict], start_date: date_type, end_date: date_type) -> list[dict]`** helper:
       - Iterate from start_date to end_date inclusive using timedelta(days=1)
       - For each day, look up day_str (YYYY-MM-DD) in db_rows dict
       - Return list of dicts with keys: date, total, up_votes, down_votes
       - Days with no data get zero counts

    6. **`_parse_date_range(arguments: dict) -> tuple[datetime, datetime, str, str]`** helper (shared by both handlers):
       - Default: last 7 days from now (UTC)
       - Parse arguments["start_date"] and arguments["end_date"] if provided using datetime.fromisoformat with Z replacement (same pattern as Phase 26)
       - Enforce 90-day max range, raise RuntimeError if exceeded
       - Raise RuntimeError if start_date >= end_date
       - Return (start_dt, end_dt, start_str, end_str) where str values are formatted as "%Y-%m-%dT%H:%M:%SZ"

    7. **`_get_feedback_summary_handler(arguments: dict[str, Any], client) -> dict[str, Any]`**:
       - Async function following the established handler signature `(arguments, client)` — `client` is unused but required by dispatch table
       - Read `ATLAS_DB_PATH` from env via `os.environ.get("ATLAS_DB_PATH", "").strip()`
       - If empty, return `{"error": "ATLAS_DB_PATH is not configured. Set this environment variable to the Atlas database path."}`
       - Parse date range using `_parse_date_range(arguments)`
       - Define inner `_query() -> dict` function containing all blocking sqlite3 work:
         - Open connection via `_open_ro(db_path)` — catch Exception and return `{"error": f"Cannot open database: {exc}"}`
         - Execute SUMMARY_SQL with (start_str, end_str) params
         - Handle NULL from SUM: use `row["up_votes"] or 0` pattern for all SUM columns
         - Compute satisfaction_pct: `round(100 * up / total, 1)` if total > 0, else None
         - Execute TREND_SQL with (start_str, end_str) params
         - Build trend_by_day dict keyed by day string
         - Call _zero_fill_trend with start_dt.date() and end_dt.date()
         - Close connection in finally block
         - Return dict with: total_votes, up_votes, down_votes, satisfaction_pct, comment_count, daily_trend, date_range (start/end ISO strings)
       - `return await asyncio.to_thread(_query)` — MUST await

    8. **`_get_low_rated_responses_handler(arguments: dict[str, Any], client) -> dict[str, Any]`**:
       - Async function, same signature pattern
       - Read ATLAS_DB_PATH from env, return error dict if empty
       - Parse date range using `_parse_date_range(arguments)`
       - Parse limit: `int(arguments.get("limit") or 20)`, clamp to range 1-100
       - Define inner `_query() -> dict`:
         - Open read-only connection via `_open_ro(db_path)`, catch Exception
         - Execute LOW_RATED_SQL with (start_str, end_str, limit)
         - Build entries list: each entry has timestamp, thread_name, comment (None if empty), has_comment (bool)
         - Close connection in finally block
         - Return dict with: entries, count (len of entries), date_range (start/end ISO strings)
       - `return await asyncio.to_thread(_query)`

    The `client` parameter type hint should be `Any` (not ExchangeClient) to avoid importing the Exchange client in this module. Add a comment: `# client is unused — feedback analytics reads SQLite directly, not Exchange Online`.
