2. **SQL constant `TOOL_FEEDBACK_SQL`** (after the existing `LOW_RATED_SQL`):
```python
TOOL_FEEDBACK_SQL = """
    SELECT
        f.thread_id,
        f.assistant_message_idx,
        f.vote,
        f.created_at,
        f.comment,
        m.messages_json,
        t.name AS thread_name
    FROM feedback f
    JOIN messages m ON m.thread_id = f.thread_id
    JOIN threads t ON t.id = f.thread_id
    WHERE f.created_at >= ? AND f.created_at <= ?
"""
```

3. **`_find_tool_names(messages, assistant_message_idx)` function** (in the Private helpers section):
- Walks the `messages` list (parsed JSON array) to find the content-bearing assistant message at the given ordinal index.
- Content-bearing = `role == "assistant"` AND `msg.get("content")` is truthy (not None, not empty string).
- Once found, walks backward to find the closest preceding assistant message with `tool_calls` or `function_call`.
- For `tool_calls` format: extracts `tc["function"]["name"]` for each tool call.
- For legacy `function_call` format: extracts `msg["function_call"]["name"]` as a single-element list.
- Stops walking backward when hitting a `user` role message (no tools used for that turn).
- Returns `[]` if no tools found.

4. **`_get_feedback_by_tool_handler(arguments, client)` async function** (after `_get_low_rated_responses_handler`):
- Same ATLAS_DB_PATH check and `_parse_date_range` pattern as existing handlers.
- Has two modes controlled by `tool_name` parameter:
  - **No tool_name (breakdown mode):** Fetches all feedback rows in date range via `TOOL_FEEDBACK_SQL`. For each row, parses `messages_json` (cache by `thread_id` to avoid reparsing), calls `_find_tool_names`, fans out vote to each tool. Aggregates per-tool: `up_votes`, `down_votes`, `total_votes`, `satisfaction_pct` (round to 1 decimal, None if total==0), `low_confidence` (True if total < 5). Sorts by `satisfaction_pct ASC` (worst first), with `low_confidence` tools sorted last. Returns `{"tools": [...], "date_range": {...}}`.
  - **With tool_name (drill-down mode):** Same fetch + correlation, but filters to rows matching the given `tool_name`. Returns the low-rated (vote='down') examples for that tool: `timestamp`, `comment` (or null), `thread_name`. Respects `limit` parameter (default 10, max 50, min 1). Returns `{"tool_name": "...", "examples": [...], "count": N, "date_range": {...}}`.
- Wrap `json.loads` in try/except `json.JSONDecodeError` — skip corrupt rows silently.
- All database I/O wrapped in `asyncio.to_thread` (same pattern as existing handlers).
