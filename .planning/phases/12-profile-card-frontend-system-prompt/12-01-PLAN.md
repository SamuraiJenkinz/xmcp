  <name>Task 1: addProfileCard function and processLine branch in app.js</name>
  <files>chat_app/static/app.js</files>
  <action>
Three changes to app.js:

1. **Add `insertCard` method** to the object returned by `createAssistantMessage()` (alongside existing `addToolPanel`, `appendText`, etc.):
```javascript
insertCard: function(cardEl) {
    els.content.insertBefore(cardEl, textNode);
    scrollToBottom();
},
```

2. **Add `addProfileCard()` function** as a standalone function (place it near `createAssistantMessage`, before `showWelcomeMessage`). The function:
   - Accepts `(assistantMsg, resultJson)` where `resultJson` is the raw `event.result` string (double-serialized JSON)
   - Calls `JSON.parse(resultJson)` wrapped in try/catch — silently returns on parse failure
   - Guards: returns early if `!profile || typeof profile !== 'object' || !profile.name`
   - Creates a `div.profile-card` container with horizontal layout
   - Creates `img.profile-card-photo` with `src = profile.photo_url + '?name=' + encodeURIComponent(profile.name)`. Sets `alt = profile.name`. If `photo_url` is falsy, still set src to empty string (proxy handles all fallbacks)
   - Creates `div.profile-card-info` containing:
     - `div.profile-card-name` with `textContent = profile.name`
     - `div.profile-card-field` with `textContent = profile.jobTitle` (only if truthy)
     - `div.profile-card-field.profile-card-dept` with `textContent = profile.department` (only if truthy)
     - `a.profile-card-email` with `href = 'mailto:' + profile.email` and `textContent = profile.email` (only if truthy)
   - Calls `assistantMsg.insertCard(card)` at the end
   - Uses `var` declarations (ES5 style matching the rest of app.js)

3. **Modify the `processLine` tool event branch** (around line 309-320). Replace the unconditional `addToolPanel` call with a conditional:
   - If `event.name === 'get_colleague_profile' && event.status === 'success'`: call `addProfileCard(assistantMsg, event.result || '')` and set `activeChip = null`
   - Else: existing `addToolPanel` call (unchanged)
   - The `removeDots()` and `markToolDone(activeChip)` calls before the branch remain unchanged

Key pitfalls to avoid:
- Do NOT access `event.result.name` directly — it's a JSON string, must parse first
- Do NOT call both `addToolPanel` and `addProfileCard` for the same event — exclusive branch
- Do NOT forget `?name=` on the photo URL — without it, proxy returns generic placeholder instead of initials
- Use `encodeURIComponent` (not `encodeURI`) for the name param
