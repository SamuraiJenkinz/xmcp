  <name>Task 1: Implement ExchangeClient class with CBA auth, script templating, and retry logic</name>
  <files>exchange_mcp/exchange_client.py</files>
  <action>
    Create `exchange_mcp/exchange_client.py` with:

    1. **Module-level PS script templates** (string constants):

       `_PS_CONNECT_TEMPLATE`:
       ```powershell
       Import-Module ExchangeOnlineManagement -ErrorAction Stop
       Connect-ExchangeOnline `
           -CertificateThumbPrint $env:AZURE_CERT_THUMBPRINT `
           -AppID $env:AZURE_CLIENT_ID `
           -Organization $env:AZURE_TENANT_DOMAIN `
           -ShowBanner:$false `
           -SkipLoadingFormatData
       ```

       `_PS_DISCONNECT`:
       ```powershell
       Disconnect-ExchangeOnline -Confirm:$false -ErrorAction SilentlyContinue
       ```

    2. **ExchangeClient class**:

       `__init__(self, timeout: int = 60, max_retries: int = 3, default_result_size: int = 100)`:
       - Store timeout, max_retries, default_result_size as instance attributes
       - Call `_verify_env()` to fail-fast on missing credentials
       - Env vars required: AZURE_CERT_THUMBPRINT, AZURE_CLIENT_ID, AZURE_TENANT_DOMAIN

       `_verify_env(self) -> None`:
       - Check all three env vars are set and non-empty
       - Raise `EnvironmentError` with list of missing vars if any are missing

       `_build_cmdlet_script(self, cmdlet_line: str) -> str`:
       - Compose a full PowerShell script:
         ```
         {preamble from ps_runner.build_script}
         try {{
             {_PS_CONNECT_TEMPLATE}
             $result = {cmdlet_line} | ConvertTo-Json -Depth 10
             Write-Output $result
         }}
         catch {{
             $errorObj = @{{ error = $_.Exception.Message; type = $_.Exception.GetType().FullName }}
             Write-Output ($errorObj | ConvertTo-Json)
             exit 1
         }}
         finally {{
             {_PS_DISCONNECT}
         }}
         ```
       - Use `ps_runner.build_script()` to prepend the UTF-8 preamble
       - IMPORTANT: The `ConvertTo-Json -Depth 10` is hardcoded in the template, not optional
       - IMPORTANT: `Disconnect-ExchangeOnline` is in the finally block, ensuring cleanup on ALL paths

       `async def run_cmdlet(self, cmdlet_line: str) -> Any`:
       - Build the full script via `_build_cmdlet_script(cmdlet_line)`
       - Call `ps_runner.run_ps(script, timeout=self.timeout)`
       - Parse the raw string output with `json.loads()`
       - If the parsed result is a dict with an "error" key, raise `RuntimeError` with the error message
       - Return the parsed JSON (dict or list)

       `async def run_cmdlet_with_retry(self, cmdlet_line: str) -> Any`:
       - Wrap `run_cmdlet()` with retry logic:
       - Up to `self.max_retries` attempts
       - On `TimeoutError` or `RuntimeError` where the error message contains throttling indicators ("throttl", "busy", "too many", "exceeded the maximum number of connections"): wait `2 ** attempt` seconds, then retry
       - On `RuntimeError` where the error indicates auth failure ("authentication", "certificate", "unauthorized", "access denied") or invalid input ("not found", "couldn't find", "ambiguous"): raise immediately, do NOT retry
       - On other `RuntimeError`: retry (assume transient)
       - Use `asyncio.sleep()` for backoff delay
       - If all retries exhausted, raise the last exception

       `async def verify_connection(self) -> bool`:
       - Run `Get-OrganizationConfig | Select-Object Name` via `run_cmdlet()`
       - Return True if result has a non-empty "Name" key
       - On any exception, return False (do not raise — this is a health check)

    3. **Error classification helper** (private):

       `_is_retryable(error_message: str) -> bool`:
       - Return False if message contains auth/input keywords (see above)
       - Return True otherwise

    Design decisions to make (Claude's Discretion from CONTEXT.md):
    - **Return type**: Return raw dicts from `run_cmdlet()`. MCP tool handlers will shape these as needed for tool responses. Raw dicts are the most flexible for the tool pattern.
    - **Error representation**: Raise exceptions from `run_cmdlet()` (RuntimeError for PS errors, TimeoutError for timeouts, json.JSONDecodeError for parse failures). MCP tool handlers will catch and wrap into isError responses.
    - **Logging**: Use Python `logging` module. Log stderr output at WARNING level even on successful PowerShell calls (PS sends warnings to stderr). Log retry attempts at INFO level.
