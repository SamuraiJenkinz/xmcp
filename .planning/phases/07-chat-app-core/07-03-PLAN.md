**Client initialization:**

The MMC CHATGPT_ENDPOINT is a full URL ending in `/chat/completions`. The OpenAI SDK's `AzureOpenAI` appends `/openai/deployments/{model}/chat/completions` automatically, which would double-append. Two approaches to handle this:

Approach A (preferred): Use `openai.OpenAI` (not AzureOpenAI) with a custom `base_url`. Strip `/chat/completions` from the endpoint to get the base deployment URL. The SDK appends `/chat/completions` itself.

Approach B (fallback): Use `openai.AzureOpenAI` with `azure_endpoint` set to the gateway base and `azure_deployment` set to the model name.

Implement Approach A first since the MMC gateway URL format doesn't match standard Azure OpenAI patterns:

```python
import logging
from openai import OpenAI
from chat_app.config import Config

logger = logging.getLogger(__name__)

_client: OpenAI | None = None

def _get_base_url() -> str:
    """Derive SDK base_url from CHATGPT_ENDPOINT.

    The env var is the full path ending in /chat/completions.
    The SDK appends /chat/completions automatically, so strip it.
    """
    endpoint = Config.CHATGPT_ENDPOINT
    suffix = "/chat/completions"
    if endpoint.endswith(suffix):
        return endpoint[: -len(suffix)]
    return endpoint

def init_openai() -> None:
    """Initialize the OpenAI client. Called once at app startup."""
    global _client
    base_url = _get_base_url()
    _client = OpenAI(
        base_url=base_url,
        api_key=Config.AZURE_OPENAI_API_KEY,
        default_headers={
            "api-key": Config.AZURE_OPENAI_API_KEY,  # MMC gateway may expect this header
        },
    )
    logger.info("OpenAI client initialized (base_url=%s)", base_url)

def get_client() -> OpenAI:
    """Return the initialized OpenAI client."""
    if _client is None:
        raise RuntimeError("OpenAI client not initialized — call init_openai() first")
    return _client
```

**Message conversion helper:**
```python
def _message_to_dict(message) -> dict:
    """Convert an OpenAI SDK ChatCompletionMessage to a plain dict.

    SDK objects are NOT JSON-serializable. We must convert to plain dicts
    before storing in session or passing to subsequent API calls.
    """
    result = {
        "role": message.role,
        "content": message.content,
    }
    if hasattr(message, "tool_calls") and message.tool_calls:
        result["tool_calls"] = [
            {
                "id": tc.id,
                "type": "function",
                "function": {
                    "name": tc.function.name,
                    "arguments": tc.function.arguments,
                },
            }
            for tc in message.tool_calls
        ]
    return result
```

**Basic chat completion (no tools yet):**
```python
def chat_completion(messages: list[dict]) -> dict:
    """Run a single chat completion (no tools, no streaming).

    Args:
        messages: Conversation history as plain dicts.

    Returns:
        Assistant message as a plain dict with 'role' and 'content'.
    """
    client = get_client()
    response = client.chat.completions.create(
        model=Config.OPENAI_MODEL,
        messages=messages,
    )
    assistant_msg = response.choices[0].message
    return _message_to_dict(assistant_msg)
```
