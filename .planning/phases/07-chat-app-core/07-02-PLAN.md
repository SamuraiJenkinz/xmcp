  <name>Task 1: Create auth.py with MSAL auth code flow routes</name>
  <files>chat_app/auth.py</files>
  <action>
Create chat_app/auth.py as a Flask Blueprint with MSAL auth code flow implementation.

**Blueprint:** `auth_bp = Blueprint("auth", __name__)`

**Helper functions:**

1. `_build_msal_app(cache=None)` — Creates `msal.ConfidentialClientApplication` using `Config.AZURE_CLIENT_ID`, `Config.AZURE_CLIENT_SECRET`, `Config.AZURE_AUTHORITY`, and optional `token_cache`.

2. `_load_cache()` — Creates `msal.SerializableTokenCache()`, deserializes from `session.get("token_cache")` if present, returns cache.

3. `_save_cache(cache)` — If `cache.has_state_changed`, serializes to `session["token_cache"]`.

4. `get_token_silently()` — Loads cache, builds MSAL app with cache, calls `cca.get_accounts()`, if accounts exist calls `cca.acquire_token_silent(["User.Read"], account=accounts[0])`, saves cache, returns result or None.

**Routes:**

1. `GET /login` — Calls `_build_msal_app().initiate_auth_code_flow(scopes=["User.Read"], redirect_uri=url_for("auth.auth_callback", _external=True))`. Stores the flow dict in `session["auth_flow"]`. Redirects to `flow["auth_uri"]`.

2. `GET /auth/callback` — Loads cache, builds MSAL app with cache, calls `app_msal.acquire_token_by_auth_code_flow(session.pop("auth_flow", {}), request.args)`. Wraps in try/except ValueError (CSRF protection) — redirects to login on failure. If `"error"` in result, renders an error page with `result.get("error_description")`. Otherwise stores `result.get("id_token_claims")` in `session["user"]`, saves cache, redirects to `/chat`.

3. `GET /logout` — Clears session with `session.clear()`, redirects to `/`.

**Auth decorator (for protecting routes):**

```python
from functools import wraps

def login_required(f):
    """Decorator to require authentication on a route."""
    @wraps(f)
    def decorated(*args, **kwargs):
        if not session.get("user"):
            return redirect(url_for("index"))
        return f(*args, **kwargs)
    return decorated
```

Important implementation notes:
- Use `from chat_app.config import Config` for all config values
- Scopes: `["User.Read"]` — minimal scope for user profile info
- The `id_token_claims` dict contains `name`, `preferred_username` (email), `oid`, `tid` — store the whole thing in session["user"]
- Handle `interaction_required` error in Conditional Access by checking if result contains error and error is "interaction_required" — redirect to login with a fresh flow
- Do NOT use the `identity` community package — use `msal` directly as documented in research
