  <name>Task 1: Migrate chat_stream from session to SQLite with auto-naming</name>
  <files>chat_app/chat.py</files>
  <action>
Rewrite chat.py to use SQLite instead of Flask session for conversation storage. Key changes:

1. REMOVE the _CONVERSATION_KEY constant entirely.
2. ADD imports: from chat_app.db import get_db, import json (for json.dumps/loads)
3. ADD import: import datetime (for fallback name)

4. ADD helper function _auto_name(text: str, max_chars: int = 30) -> str:
   - Strip text. If len <= max_chars, return as-is.
   - Otherwise return text[:max_chars].rstrip() + "\u2026" (ellipsis character)

5. ADD helper function _fallback_name() -> str:
   - now = datetime.datetime.now()
   - On Windows use %#d and %#I for no-leading-zero (the server runs on Windows per project constraints)
   - Return now.strftime("Chat \u2014 %b %#d, %#I:%M %p")
   - This produces "Chat — Mar 21, 2:30 PM"

6. MODIFY chat_stream route:
   - POST body now requires thread_id in addition to message:
     thread_id = (request.json or {}).get("thread_id")
     If thread_id is None, return error SSE: {"type": "error", "message": "No thread selected"}

   - Read user_id from session BEFORE generator (same pre-read pattern):
     user_info = session.get("user") or {}
     username = user_info.get("preferred_username", "unknown")
     user_id = user_info.get("oid", "")

   - Validate thread ownership BEFORE generator entry:
     db = get_db()
     thread_row = db.execute(
         "SELECT id, name FROM threads WHERE id = ? AND user_id = ?",
         (thread_id, user_id),
     ).fetchone()
     If thread_row is None, return error SSE: {"type": "error", "message": "Thread not found"}

   - Load conversation from SQLite (replaces session.get(_CONVERSATION_KEY)):
     msg_row = db.execute(
         "SELECT messages_json FROM messages WHERE thread_id = ?",
         (thread_id,),
     ).fetchone()
     conversation = json.loads(msg_row["messages_json"]) if msg_row else []
     if not conversation:
         conversation = [build_system_message()]

   - Capture thread_name BEFORE generator for auto-naming check:
     thread_name = thread_row["name"] if thread_row else ""

   - Inside generate() — AFTER full_response is assembled and conversation is updated
     (step 6 in current code), replace the session write with SQLite writes:

     # Save conversation to SQLite
     db = get_db()
     db.execute(
         "UPDATE messages SET messages_json = ? WHERE thread_id = ?",
         (json.dumps(conversation), thread_id),
     )

     # Auto-name thread if this is the first message (name is empty)
     if not thread_name:
         auto = _auto_name(user_message) if user_message.strip() else _fallback_name()
         db.execute(
             "UPDATE threads SET name = ?, updated_at = strftime('%Y-%m-%dT%H:%M:%SZ','now') WHERE id = ?",
             (auto, thread_id),
         )
     else:
         # Touch updated_at so thread sorts to top
         db.execute(
             "UPDATE threads SET updated_at = strftime('%Y-%m-%dT%H:%M:%SZ','now') WHERE id = ?",
             (thread_id,),
         )
     db.commit()

     DELETE the two lines: session[_CONVERSATION_KEY] = conversation / session.modified = True

   - In the SSE "done" event, include the auto-generated thread name so the frontend
     can update the sidebar without re-fetching:
     If auto-naming happened (thread_name was empty), yield:
       _sse({"type": "thread_named", "thread_id": thread_id, "name": auto})
     before the final done event.

7. REMOVE the /chat/clear route entirely (both the function and the @chat_bp.route decorator).
   The sidebar delete button in Plan 03 replaces this functionality.

8. UPDATE the module docstring to reflect the new thread-based architecture. Remove
   references to session["conversation"] and /chat/clear.

CRITICAL NOTES:
- get_db() is safe inside stream_with_context generators (Flask request context is preserved).
- db.commit() MUST be called after writes — close_db does NOT commit.
- The thread_id ownership check uses AND user_id = ? — never trust frontend-supplied IDs alone.
- Keep the existing pre-read pattern: all session data read BEFORE entering generate().
