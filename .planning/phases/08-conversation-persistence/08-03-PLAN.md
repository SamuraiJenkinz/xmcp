Rewrite app.js to add thread management while preserving all existing chat streaming logic.

ADD the following state and functions at the top of the IIFE (after DOM references):

1. STATE VARIABLES:
   var currentThreadId = null;  // Active thread ID
   var sidebarEl = document.getElementById('sidebar');
   var threadListEl = document.getElementById('thread-list');
   var newChatBtn = document.getElementById('new-chat-btn');

   Early exit if not on chat page: check for all required elements.

2. FUNCTION fetchThreads():
   - fetch('/api/threads').then(r => r.json()).then(renderThreadList)
   - Call on page load

3. FUNCTION renderThreadList(threads):
   - Clear threadListEl innerHTML
   - For each thread:
     - Create <li class="thread-item"> with data-id attribute
     - If thread.id === currentThreadId, add "active" class
     - Create <span class="thread-name" contenteditable="true"> with text = thread.name || "New chat"
       Set span.dataset.originalName = thread.name || ""
     - Create <button class="thread-delete" title="Delete"> with text content = unicode multiply sign (x)
     - Wire click on li (excluding name span in edit mode and delete button):
       li.addEventListener('click', function(e) {
         if (e.target.classList.contains('thread-name') && document.activeElement === e.target) return;
         if (e.target.classList.contains('thread-delete')) return;
         switchThread(thread.id);
       });
     - Wire delete button click (with stopPropagation):
       delBtn.addEventListener('click', function(e) {
         e.stopPropagation();
         deleteThread(thread.id);
       });
     - Wire inline rename on the name span using makeRenameHandler(thread.id, nameSpan)
     - Append children and li to threadListEl

4. FUNCTION switchThread(threadId):
   - If threadId === currentThreadId, return (already active)
   - Set currentThreadId = threadId
   - Clear messagesEl innerHTML (remove all message bubbles)
   - Re-add the welcome message if thread has no messages (or always show it — see below)
   - Fetch thread messages: fetch('/api/threads/' + threadId + '/messages')
     .then(r => r.json())
     .then(function(data) {
       var messages = data.messages || [];
       // Filter out system messages — don't display those
       messages.forEach(function(msg) {
         if (msg.role === 'user') {
           appendUserMessage(msg.content);
         } else if (msg.role === 'assistant' && msg.content) {
           // Render as completed assistant message (no streaming cursor)
           var els = createMessageEl('assistant');
           var p = document.createElement('p');
           p.textContent = msg.content;
           els.content.appendChild(p);
           messagesEl.appendChild(els.wrapper);
         }
         // Skip tool/function/system messages in display
       });
       // If no non-system messages, show the welcome message
       if (messages.filter(function(m) { return m.role !== 'system'; }).length === 0) {
         showWelcomeMessage();
       }
       scrollToBottom();
     });
   - Update sidebar active state:
     var items = threadListEl.querySelectorAll('.thread-item');
     items.forEach(function(item) {
       item.classList.toggle('active', parseInt(item.dataset.id) === threadId);
     });

5. FUNCTION createNewThread():
   - fetch('/api/threads', { method: 'POST' })
     .then(r => r.json())
     .then(function(data) {
       currentThreadId = data.id;
       fetchThreads();  // Re-render sidebar
       // Clear chat area and show welcome message
       messagesEl.innerHTML = '';
       showWelcomeMessage();
       inputEl.focus();
     });

6. FUNCTION deleteThread(threadId):
   - if (!window.confirm('Delete this conversation? This cannot be undone.')) return;
   - fetch('/api/threads/' + threadId, { method: 'DELETE' })
     .then(function() {
       // If we deleted the active thread, switch to another
       if (threadId === currentThreadId) {
         currentThreadId = null;
       }
       // Re-fetch thread list; after render, select the first available thread
       fetch('/api/threads').then(function(r) { return r.json(); })
         .then(function(threads) {
           renderThreadList(threads);
           if (threads.length > 0 && !currentThreadId) {
             switchThread(threads[0].id);
           } else if (threads.length === 0) {
             // No threads left — create a new one
             createNewThread();
           }
         });
     });

7. FUNCTION makeRenameHandler(threadId, nameEl):
   - On blur: send PATCH with new name. Update nameEl.dataset.originalName on success.
   - On keydown Enter: preventDefault, blur (triggers the blur handler)
   - On keydown Escape: restore from nameEl.dataset.originalName, blur
   - On focus: select all text for easy replacement (window.getSelection / Range API)

8. FUNCTION showWelcomeMessage():
   - Recreate the welcome message HTML (same as the Jinja template) and append to messagesEl
   - Wire example query buttons within the welcome message

9. MODIFY sendMessage(text):
   - If no currentThreadId, create a new thread first, then send:
     If currentThreadId is null, call createNewThread and chain the send after the thread is created.
     Alternatively: always create a thread via New Chat before the first message.
     RECOMMENDED approach: if currentThreadId is null, POST /api/threads first, set currentThreadId from response, then proceed with the fetch to /chat/stream.

   - In the fetch to /chat/stream, include thread_id in the POST body:
     body: JSON.stringify({ message: text, thread_id: currentThreadId })

   - After streaming completes (in the "done" event handler), re-fetch the thread list
     to update sidebar ordering (the thread's updated_at changed).

   - Handle the new "thread_named" SSE event type: when received, update the thread name
     in the sidebar without a full re-fetch. Find the thread-item with matching data-id
     and update its .thread-name textContent.

10. INITIAL LOAD LOGIC (at bottom of IIFE, after all function definitions):
    - Read lastThreadId from the data attribute:
      var layoutEl = document.querySelector('.app-layout');
      var lastThreadId = layoutEl ? parseInt(layoutEl.dataset.lastThreadId) : null;
    - Fetch threads list. On response:
      If threads.length > 0:
        renderThreadList(threads)
        switchThread(lastThreadId && threads.some(t => t.id === lastThreadId) ? lastThreadId : threads[0].id)
      Else:
        createNewThread()  // First visit — auto-create a thread

11. WIRE newChatBtn click:
    newChatBtn.addEventListener('click', createNewThread);

12. PRESERVE all existing functionality:
    - Auto-expanding textarea (resizeInput, resetInputSize)

    - Streaming state management (setStreaming)
    - Keyboard shortcut (Ctrl+Enter)
    - Example query buttons (delegate handler on messagesEl — may need re-wiring since
      messagesEl content changes on thread switch; use event delegation which already works)
