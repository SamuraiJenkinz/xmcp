Create chat_app/db.py following the official Flask tutorial pattern exactly:
- get_db() -> sqlite3.Connection: stores connection in flask.g, sets row_factory=sqlite3.Row, executes PRAGMA journal_mode=WAL and PRAGMA foreign_keys=ON on first open. Uses current_app.config["DATABASE"] for the path. Uses detect_types=sqlite3.PARSE_DECLTYPES.
- close_db(e=None): pops db from g and closes if present.
- init_db(): opens schema.sql via current_app.open_resource() and executescript. Also auto-called inside get_db() if the database file does not exist yet — check os.path.exists(current_app.config["DATABASE"]) before returning from get_db(), and if the file was just created (i.e., this is the first connect), call init_db() to bootstrap the schema. This ensures the app self-bootstraps on first startup without requiring a manual CLI command.
- init_db_command(): click command "init-db" that calls init_db() and prints confirmation.
- init_app(app): registers teardown_appcontext(close_db) and cli.add_command(init_db_command).

Create chat_app/schema.sql:
```sql
PRAGMA journal_mode=WAL;
PRAGMA foreign_keys=ON;

CREATE TABLE IF NOT EXISTS threads (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id    TEXT    NOT NULL,
    name       TEXT    NOT NULL DEFAULT '',
    created_at TEXT    NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now')),
    updated_at TEXT    NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now'))
);

CREATE TABLE IF NOT EXISTS messages (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    thread_id     INTEGER NOT NULL REFERENCES threads(id) ON DELETE CASCADE,
    messages_json TEXT    NOT NULL DEFAULT '[]'
);

CREATE INDEX IF NOT EXISTS idx_threads_user_updated
    ON threads(user_id, updated_at DESC);
```

<task type="auto">
  <name>Task 2: Create thread CRUD blueprint and register in app factory</name>
  <files>chat_app/conversations.py, chat_app/app.py</files>
  <action>
Create chat_app/conversations.py as a Flask Blueprint named "conversations_bp":

Helper function _user_id() -> str: returns session["user"]["oid"] (Azure AD object ID). Falls back to empty string if not present.

5 routes, ALL decorated with @login_required, ALL using parameterized queries with user_id ownership checks:

1. GET /api/threads — list_threads():
   - Query: SELECT id, name, updated_at FROM threads WHERE user_id = ? ORDER BY updated_at DESC
   - Return: jsonify([dict(r) for r in rows])

2. POST /api/threads — create_thread():
   - INSERT INTO threads (user_id, name) VALUES (?, '')
   - INSERT INTO messages (thread_id, messages_json) VALUES (?, '[]')
   - db.commit()
   - Return: jsonify({"id": thread_id, "name": ""}), 201

3. GET /api/threads/<int:thread_id>/messages — get_messages():
   - Verify ownership: SELECT id FROM threads WHERE id = ? AND user_id = ?
   - If not found: return jsonify({"error": "Not found"}), 404
   - Query: SELECT messages_json FROM messages WHERE thread_id = ?
   - Return: jsonify({"messages": json.loads(row["messages_json"]) if row else []})

4. PATCH /api/threads/<int:thread_id> — rename_thread():
   - Read new_name from request JSON body, strip and truncate to 100 chars
   - UPDATE threads SET name = ?, updated_at = strftime(...) WHERE id = ? AND user_id = ?
   - db.commit()
   - Return: jsonify({"id": thread_id, "name": new_name})

5. DELETE /api/threads/<int:thread_id> — delete_thread():
   - DELETE FROM threads WHERE id = ? AND user_id = ?
   - db.commit() (CASCADE deletes messages row)
   - Return: jsonify({"deleted": True})

CRITICAL: Every query that touches threads or messages MUST include AND user_id = ? for ownership check. Convert sqlite3.Row to dict() before jsonify().
